# 📅 캘린더 디자인 개선 기획서

**간호사 팀 스케줄링 특화 캘린더 UX/UI 개선안**

---

## 📊 현재 상태 분석

### 현재 구현된 뷰
1. **주간 뷰 (Week View)**
   - 그리드 형태: 팀원 × 7일
   - 교대근무 타입별 색상 표시
   - 개인 일정 표시
   - 셀 클릭으로 일정 추가/수정

2. **월간 뷰 (Month View)**
   - 전통적인 캘린더 그리드 (7×5/6)
   - 날짜별 일정 표시
   - 팀원별 색상 구분

3. **연간 뷰 (Year View)**
   - 월별 통계 카드
   - 교대근무 통계 (데이/이브닝/나이트/휴무)
   - 월 클릭 시 월간 뷰로 이동

### 현재 디자인 시스템
- **토스 스타일**: Glass Morphism, 부드러운 애니메이션
- **색상 시스템**: 교대근무 타입별 고정 색상
- **레이아웃**: `glass-card`, `rounded-3xl`, `toss-shadow`

## 🔍 주요 캘린더 앱 비교 분석

### 1. Google Calendar

**뷰 구조:**
- **일간 (Day)**: 시간대별 상세 일정
- **주간 (Week)**: 시간대 × 7일 그리드
- **월간 (Month)**: 날짜별 일정 요약
- **연간 (Year)**: 월별 미리보기

**특징:**
- ✅ 시간대 기반 주간 뷰 (시간대별 상세 표시)
- ✅ 드래그 앤 드롭으로 일정 이동
- ✅ 색상 코딩으로 카테고리 구분
- ✅ 일정 오버플로우 처리 (더보기 버튼)

**우리 앱과의 차이점:**
- 우리는 팀원 × 날짜 그리드 (Google은 시간 × 날짜)
- 우리는 교대근무 중심, Google은 시간 중심

### 2. Notion Calendar

**뷰 구조:**
- **주간 (Week)**: 시간대별 상세 뷰
- **월간 (Month)**: 미니멀한 그리드
- **일간 (Day)**: 상세 일정 뷰

**특징:**
- ✅ 미니멀한 디자인
- ✅ 빠른 일정 추가
- ✅ 간단한 색상 구분

**우리 앱과의 차이점:**
- Notion은 개인 일정 중심, 우리는 팀 일정 중심

### 3. Apple Calendar

**뷰 구조:**
- **일간/주간/월간**: 전통적인 캘린더 구조
- **목록 (List)**: 시간순 정렬

**특징:**
- ✅ 직관적인 네비게이션
- ✅ 일정 오버플로우 처리
- ✅ 색상 코딩

### 4. 간호사 스케줄링 앱 (NurseGrid, ShiftMed)

**특징:**
- ✅ **주간 뷰 중심**: 간호사는 주 단위로 계획
- ✅ **팀원별 그리드**: 팀원 × 날짜 구조
- ✅ **교대근무 색상**: Day/Evening/Night 구분
- ✅ **빠른 일정 추가**: 드래그 앤 드롭
- ✅ **시프트 교환 기능**: 팀원 간 시프트 교환

## 🎯 간호사 업무 특성 분석

### 핵심 요구사항

1. **교대근무 패턴 관리**
   - Day (07:00-15:00)
   - Evening (15:00-23:00)
   - Night (23:00-07:00)
   - Off (휴무)
   - **→ 색상으로 즉시 인식 가능해야 함**

2. **팀 스케줄 확인**
   - 누가 언제 근무하는지 한눈에 파악
   - 팀원 간 근무 분배 확인
   - **→ 그리드 형태가 최적**

3. **빠른 일정 조정**
   - 긴급 변경 대응
   - 시프트 교환 요청
   - **→ 드래그 앤 드롭 또는 빠른 추가**

4. **장기 계획**
   - 월별/연간 근무 패턴 확인
   - 통계 및 분석
   - **→ 월간/연간 뷰 필요**

## 💡 개선 방향 제안

### 1. 뷰 구조 검토

#### ✅ 유지할 뷰
- **주간 뷰**: 간호사 업무 특성상 가장 중요
- **월간 뷰**: 장기 계획 및 전체 파악
- **연간 뷰**: 통계 및 패턴 분석

#### 🤔 검토 필요
- **일간 뷰 추가?**: Google Calendar처럼 시간대별 상세 뷰
  - **장점**: 하루 일정 상세 관리
  - **단점**: 간호사는 교대근무 단위라 시간대별 세분화 필요성 낮음
  - **결론**: 현재 주간 뷰로 충분, 일간 뷰는 선택사항

### 2. 주간 뷰 개선안

#### 현재 문제점
- 셀 높이가 고정되어 일정이 많으면 가려짐
- 일정 추가 버튼이 hover 시에만 표시
- 스크롤이 없어 많은 팀원 표시 어려움

#### 개선 제안

**A안: 스크롤 가능한 그리드**
```
┌─────────┬─────┬─────┬─────┐
│ 팀원    │ 월  │ 화  │ 수  │ ← 고정 헤더
├─────────┼─────┼─────┼─────┤
│ 염정원  │ [D] │ [E] │ [N] │ ← 스크롤 가능
│ 김철수  │ [E] │ [N] │ [D] │
│ ...     │ ... │ ... │ ... │
└─────────┴─────┴─────┴─────┘
```

**B안: 시간대 추가 (Google Calendar 스타일)**
```
┌─────────┬─────┬─────┬─────┐
│ 시간    │ 월  │ 화  │ 수  │
├─────────┼─────┼─────┼─────┤
│ 07:00   │     │     │     │
│ 15:00   │     │     │     │
│ 23:00   │     │     │     │
└─────────┴─────┴─────┴─────┘
```
- **단점**: 간호사는 교대근무 단위라 시간대 세분화 불필요
- **결론**: A안 추천

**C안: 현재 구조 유지 + 개선**
- 셀 높이 자동 조절
- 일정 오버플로우 처리 (더보기 버튼)
- 항상 보이는 추가 버튼

### 3. 월간 뷰 개선안

#### 현재 문제점
- 팀원별 일정이 작은 점으로만 표시
- 여러 팀원의 일정이 겹치면 구분 어려움
- 일정 상세 정보 확인 어려움

#### 개선 제안

**A안: 팀원별 색상 강화**
- 각 날짜에 팀원별 색상 바 표시
- 교대근무 타입 아이콘 추가

**B안: 일정 오버플로우 처리**
- 일정이 많으면 "더보기" 표시
- 클릭 시 상세 모달 표시

**C안: 하이브리드**
- 주요 팀원만 색상 강조
- 나머지는 통계로 표시

### 4. 연간 뷰 개선안

#### 현재 상태
- 월별 통계 카드
- 교대근무 통계 표시
- 월 클릭 시 월간 뷰 이동

#### 개선 제안
- ✅ 현재 구조 유지 (이미 잘 되어 있음)
- 월별 근무 시간 통계 추가
- 팀원별 연간 통계 비교

## 🎨 디자인 시스템 준수 사항

### 필수 유지 요소
1. **Glass Morphism**: `glass-card`, `toss-shadow`
2. **Border Radius**: `rounded-3xl`, `rounded-2xl`
3. **색상 시스템**: 교대근무 색상 고정
4. **애니메이션**: `motion/react` 사용
5. **타이포그래피**: 기존 폰트 크기 유지

### 개선 가능 요소
1. **그리드 레이아웃**: 더 명확한 구분선
2. **일정 표시**: 더 읽기 쉬운 형태
3. **인터랙션**: 드래그 앤 드롭 추가 검토
