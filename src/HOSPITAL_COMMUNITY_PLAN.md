# 🏥 병원별 커뮤니티 기능 기획서

**병원별 구분 저장 및 커뮤니티 구축**

---

## 📋 기획안 요약

### 핵심 아이디어
1. **병원별로 구분하여 저장**
   - 사용자와 팀을 병원별로 그룹화
   - 병원 ID를 기준으로 데이터 분리

2. **하단 네비게이션에 "커뮤니티" 탭 추가**
   - 현재: 팀 스케줄, 마이 스케줄, 팀 관리
   - 추가: 커뮤니티 (팀 관리 우측)

3. **병원별 커뮤니티 구축**
   - 병원 오피셜 정보
   - 식단표
   - 블라인드 게시판 (익명 소통)

---

## 🔍 기획적 차별점 분석

### ✅ 강점 (차별화 포인트)

#### 1. **병원 단위 커뮤니티**
- **기존**: 팀 단위 게시판 (작은 그룹, 5-20명)
- **제안**: 병원 단위 커뮤니티 (큰 그룹, 수백-수천명)
- **차별점**: 더 넓은 네트워킹, 정보 공유 확대

#### 2. **병원 오피셜 정보 제공**
- **기존**: 팀 내부 공지사항만
- **제안**: 병원 전체 공지사항, 이벤트, 정책 등
- **차별점**: 병원과 직접 연계된 정보 제공

#### 3. **식단표 기능**
- **기존**: 없음
- **제안**: 병원 식당 식단표 제공
- **차별점**: 실용적이고 일상적인 정보 제공

#### 4. **블라인드 게시판**
- **기존**: 팀 게시판 (실명)
- **제안**: 익명 게시판 (블라인드 스타일)
- **차별점**: 자유로운 소통, 불만/건의사항 공유

#### 5. **통합 플랫폼**
- **기존**: 스케줄 관리만
- **제안**: 스케줄 + 커뮤니티 + 정보 제공
- **차별점**: 올인원 플랫폼으로 사용자 유지율 증가

## 🎯 기능적 차별점 분석

### ✅ 강점

#### 1. **병원별 데이터 분리**
- 병원 ID로 자동 그룹화
- 같은 병원 사용자만 접근 가능
- 보안 및 프라이버시 강화

#### 2. **다양한 콘텐츠 타입**
- 공지사항 (병원 오피셜)
- 식단표 (일일 업데이트)
- 블라인드 게시판 (익명)
- 질문/답변 게시판 (선택)

#### 3. **권한 관리**
- 병원 관리자: 공지사항, 식단표 업로드
- 일반 사용자: 블라인드 게시판 참여
- 익명성 보장

## 📊 시장 비교 분석

### 유사 서비스

#### 1. **병원 공식 앱**
- **기능**: 공지사항, 예약, 진료 정보
- **차별점**: 커뮤니티 기능 없음, 익명 소통 없음

#### 2. **블라인드 앱**
- **기능**: 익명 게시판, 직장별 커뮤니티
- **차별점**: 병원 오피셜 정보 없음, 식단표 없음

#### 3. **간호사 커뮤니티 앱**
- **기능**: 정보 공유, 질문/답변
- **차별점**: 병원별 구분 없음, 식단표 없음

### 우리의 차별점

✅ **병원별 구분**: 같은 병원 직원만 접근
✅ **오피셜 정보**: 병원 공식 정보 제공
✅ **식단표**: 실용적 일상 정보
✅ **블라인드**: 익명 소통 공간
✅ **스케줄 통합**: 스케줄 관리와 커뮤니티 통합

## 💡 개선 제안

### 1. **커뮤니티 구조**

#### 탭 구성
```
커뮤니티
├── 공지사항 (병원 오피셜)
├── 식단표
├── 블라인드 (익명 게시판)
└── 질문/답변 (선택)
```

#### 권한 관리
- **병원 관리자**: 공지사항, 식단표 업로드 권한
- **일반 사용자**: 블라인드 게시판 참여
- **익명성**: 블라인드 게시판은 완전 익명

### 2. **데이터베이스 구조**

#### 추가 테이블
```sql
-- hospital_communities: 병원별 커뮤니티
-- community_posts: 게시글 (공지사항, 블라인드 등)
-- community_comments: 댓글
-- meal_menus: 식단표
-- hospital_official_info: 병원 오피셜 정보
```

### 3. **UI/UX 제안**

#### 하단 네비게이션
- **모바일**: 4개 탭 (팀 스케줄, 마이 스케줄, 팀 관리, 커뮤니티)
- **PC**: 헤더에 탭 추가

#### 커뮤니티 뷰
- **탭 구조**: 공지사항, 식단표, 블라인드
- **필터링**: 카테고리별, 인기순, 최신순
- **검색**: 게시글 검색 기능

## 💬 최종 평가

### ✅ 기획적 차별점: **매우 강함**
- 병원별 커뮤니티는 시장에서 드문 기능
- 오피셜 정보 + 블라인드 조합은 독특함
- 식단표는 실용적이고 일상적

### ✅ 기능적 차별점: **강함**
- 병원별 데이터 분리는 보안 강화
- 다양한 콘텐츠 타입으로 활용도 증가
- 통합 플랫폼으로 사용자 유지율 증가

### ⚠️ 주의사항
- 병원 관리자 인증 시스템 필요
- 부적절한 게시글 모니터링 필요
- 확장성 고려한 데이터베이스 설계
